# Security Policy

We take the security of OpenSquawk and its users seriously.

## Reporting a vulnerability

Please report security issues **privately**. Do not open a public GitHub issue for a vulnerability.

📧 **info@opensquawk.de** — subject "Security"

Include, where possible:

- A description of the issue and its impact
- Steps to reproduce
- Affected version / commit
- Any suggested remediation

We will acknowledge your report as quickly as we can and keep you updated on the fix. Please give us a
reasonable amount of time to address the issue before any public disclosure.

## Scope

This policy covers the open-source **OpenSquawk Core** in this repository. Issues affecting the hosted
OpenSquawk Cloud, accounts or billing are handled through the same contact address and treated with the
same priority.

Thank you for helping keep OpenSquawk and its pilots safe.
