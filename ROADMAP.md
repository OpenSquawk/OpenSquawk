# OpenSquawk Roadmap

This roadmap is intentionally transparent about what is **free and open** versus what is **paid cloud**.
See [`COMMERCIAL.md`](COMMERCIAL.md) for the licensing boundary and [`open-source`](https://opensquawk.de/open-source)
for the full open/closed breakdown.

The live, votable roadmap lives at **https://opensquawk.de/roadmap**.

## Free Core (open source, AGPLv3)

- [x] Structured radio training scenarios
- [x] Readback & phraseology drills
- [ ] Basic IFR clearance training scenarios
- [ ] Local scenario runner
- [ ] Plugin API (SDK) — build your own scenarios and add-ons
- [ ] Simulator connectors (MSFS SimConnect, X-Plane, FSUIPC, SimBrief, VATSIM data)
- [ ] German phraseology notes

## Pro Cloud (commercial)

- [ ] AI debriefing after every session
- [ ] VATSIM-ready score & error analysis
- [ ] Premium voice models
- [ ] Progress tracking & cloud sync
- [ ] Premium scenario packs (VATSIM first-flight, IFR Germany, A320 IFR, emergency comms)
- [ ] Team / club features
- [ ] Founder program (first 100 accounts)

## Community contributions

Great first contributions live as GitHub issues labelled `good first issue` and `help wanted`.
Good areas to help with:

- New training scenarios (e.g. "Add EDDF IFR departure example")
- Simulator connector docs and improvements
- Phraseology notes and regional differences
- VFR pattern and beginner clearance scenarios

Anything involving the hosted cloud, AI scoring or billing is developed in private repositories and is
out of scope for community contributions.

## Not on the roadmap

- Releasing AI prompts, scoring or feedback logic
- Publishing billing / cloud infrastructure under the AGPLv3 core
- Selling "lifetime unlimited AI" — usage limits keep the platform sustainable
